📝 Prompt

# Zero-Shot Chain-of-Thought

SYSTEM:
You are a senior financial analyst. When solving
problems, always reason through each step explicitly
before giving your final answer. Show your work.

USER:
A company's revenue grew 23% YoY to $4.2M.
Their gross margin improved from 41% to 47%.
What was last year's gross profit, and by how
much (in dollars) did gross profit increase?

# Key: "show your work" triggers CoT
# No examples needed — zero-shot CoT

✅ Model Output

Step 1: Find last year's revenue.
$4.2M ÷ 1.23 = $3.415M (last year)

Step 2: Calculate last year's gross profit.
$3.415M × 41% = $1.400M

Step 3: Calculate this year's gross profit.
$4.2M × 47% = $1.974M

Step 4: Find the increase.
$1.974M - $1.400M = $574K increase

ANSWER: Last year's gross profit was $1.40M.
It increased by $574,000 (41% growth).

# Few-Shot — Sentiment + Intent Classifier

SYSTEM: Classify customer messages.
Output JSON: {sentiment, intent, urgency}

EXAMPLES:
Input: "This is the third time I've emailed!"
Output: {"sentiment":"frustrated","intent":"escalate","urgency":"high"}

Input: "Could I get an invoice copy?"
Output: {"sentiment":"neutral","intent":"billing","urgency":"low"}

Input: "Love the new feature, but one small thing..."
Output: {"sentiment":"positive","intent":"feedback","urgency":"low"}

NOW CLASSIFY:
"My account is locked and I have a demo in 1hr"

✅ Model Output

{
  "sentiment": "stressed",
  "intent": "account_access",
  "urgency": "critical"
}

---
# Why this works:
# • 3 examples cover emotional range
# • Format locked via example demonstration
# • Model extrapolates "critical" from
#   the time constraint context
# • No explicit urgency scale defined —
#   examples taught it implicitly

📝 Prompt

# Structured Output — Product Extractor

SYSTEM:
Extract product information from text.
Return ONLY valid JSON matching this schema:
{
  "name": string,
  "price_usd": number | null,
  "features": string[],
  "in_stock": boolean,
  "confidence": "high"|"medium"|"low"
}
No markdown. No explanation. JSON only.

USER:
"The Pro X Headphones retail for $299 and
come with ANC, 40hr battery, and USB-C.
Currently showing limited availability."

✅ Model Output

{
  "name": "Pro X Headphones",
  "price_usd": 299,
  "features": [
    "Active Noise Cancellation",
    "40-hour battery life",
    "USB-C charging"
  ],
  "in_stock": true,
  "confidence": "medium"
}

# Note: "limited availability" → in_stock: true
# (not false) — model correctly infers
# "limited" ≠ "out of stock"
# confidence: medium reflects ambiguity

# Multi-Agent Research System

SYSTEM (Orchestrator):
You manage a research team. Break the user's
question into subtasks. Assign each to an agent:

AGENTS:
- search_agent(query) → web results
- read_agent(url) → page summary  
- critic_agent(text) → fact-check
- synthesizer_agent(facts[]) → report

Always run critic_agent before synthesizer.
Return final report in markdown.

USER:
"Compare LLaMA 3 vs GPT-4o for enterprise use"

# Agents run in parallel where possible
# Critic gate prevents hallucinations

✅ Agent Execution Trace

ORCHESTRATOR: Splitting into 4 subtasks...

→ search_agent("LLaMA 3 enterprise")
→ search_agent("GPT-4o enterprise pricing")
   [parallel execution]

→ read_agent(meta.com/llama3)
→ read_agent(openai.com/gpt-4o)

→ critic_agent(merged_findings)
   [flags 2 unverified claims]

→ search_agent(flagged_claims) 
   [re-verification loop]

→ synthesizer_agent(verified_facts)
   OUTPUT: 800-word comparison report ✓
